    背景介绍

    在地理信息系统（GIS）工作中，地块边界文件的规范性检查是一项重要且繁琐的工作。为了提高工作效率，减少人工检查的错误率，我开发了这个Python自动化边界文件检查工具。

    工具功能特点

    🔍 主要功能

    • 批量检查：支持同时检查多个zip格式的边界文件
    • 规范性验证：检查shp文件的完整性、字段规范性、坐标系等
    • 位置校验：验证地块位置是否在边界范围内
    • 可视化展示：生成交互式地图展示检查结果
    • 结果导出：详细的Excel检查报告

    📋 检查内容

    1. 文件完整性检查

      • CPG编码文件是否存在
      • SHP几何文件是否有效
      • PRJ坐标系文件是否存在

    2. 数据规范性检查

      • 必要字段完整性（地块名称、地块代码、行政区代码等）
      • 字段内容是否为空
      • 几何类型是否为面（Polygon）

    3. 空间位置校验

      • 地块坐标点是否在边界范围内
      • 自动计算并提供正确的中心点坐标

    4. 坐标系检查

      • 坐标系类型识别（投影/地理坐标系）
      • 坐标系兼容性检查

    适用场景

    • 土地规划部门的边界文件质量检查
    • GIS数据入库前的规范性验证
    • 批量处理大量地块边界数据
    • 地理数据质量控制和审核

    技术特色

    🛠️ 核心技术栈

    • GeoPandas: 地理数据处理
    • Shapely: 几何计算和空间分析
    • Folium: 交互式地图生成
    • Pandas: 数据处理和分析
    • Tkinter: 图形用户界面

    🎯 技术亮点

    • 智能编码检测：自动识别shp文件编码（GBK/UTF-8）
    • 坐标系自适应：支持地理坐标系和投影坐标系转换
    • 中文路径支持：完美处理中文文件名和路径
    • 错误恢复机制：异常处理和临时文件清理

    输入要求

    📁 文件结构

    📊 Excel文件格式

    地块信息.xlsx必须包含以下列：

    • 地块编码：13位数字编码
    • 经度：地块中心点经度
    • 纬度：地块中心点纬度
    • 地块名称（可选）：地块描述信息

    📦 ZIP文件命名规范

    • 文件名格式：*{13位地块编码}.zip
    • 例如：初步调查4420013990012.zip
    • ZIP内必须包含完整的shp文件组（.shp, .shx, .dbf, .prj等）

    📈 检查报告

    🗺️ 可视化地图

    • 交互式HTML地图文件

    📊 统计信息

    常见问题解决

    1. 编码问题

    问题：中文文件名显示乱码
    解决：工具已集成自动编码检测，支持GBK/UTF-8自动识别

    2. 坐标系问题

    问题：坐标系不匹配导致位置检查失败
    解决：工具支持地理坐标系和投影坐标系自动转换

    3. 依赖库安装

    问题：geopandas安装困难
    解决：建议使用conda安装

    4. EXE文件过大

    问题：打包后文件超过100MB
    解决：使用--exclude-module排除不必要的模块
